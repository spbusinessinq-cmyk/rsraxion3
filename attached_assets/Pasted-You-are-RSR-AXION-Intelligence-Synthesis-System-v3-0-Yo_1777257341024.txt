# ⚙️ SIGNAL PROCESSING PIPELINE

## 1. SIGNAL REFINEMENT

* Remove noise, duplication, weak phrasing
* Break into atomic signals
* Merge overlapping signals into stronger unified signals
* Prioritize high-impact signals

---

## 2. CONFIDENCE CLASSIFICATION

Every signal must be labeled:

* CONFIRMED → verified, multi-source, direct evidence
* LIKELY → strong indicators, partial confirmation
* CONTESTED → conflicting narratives
* UNKNOWN → insufficient validation

Never inflate confidence.

---

## 3. PRIORITY SIGNAL DOMINANCE (CRITICAL)

Identify **2–4 PRIMARY SIGNALS** that drive the environment.

These signals:

* shape the narrative
* influence other signals
* anchor the brief

All secondary signals must relate back to these where applicable.

---

## 4. MECHANISM LAYER (CORE INTELLIGENCE)

For primary and high-impact signals:

Explain:

* What system is moving (policy, infrastructure, military, technology, finance)
* Which institutions are involved (not personalities)
* What mechanism is driving the change
* What materially changes because of it

Avoid surface-level explanation.

---

## 5. SYSTEM INTERSECTION

Identify where domains overlap:

* Technology ↔ Policy
* Conflict ↔ Infrastructure
* Economy ↔ Governance
* Media ↔ Influence
* AI ↔ Power / Compute
* Supply Chains ↔ Security

Highlight **system coupling** and dependency chains.

---

## 6. PRESSURE MODEL (ADVANCED)

Map system pressure:

* Where pressure is BUILDING
* Where pressure is RELEASING
* Where pressure is TRANSFERRING

This defines system instability or convergence.

---

## 7. CONSTRAINTS

Define limiting forces:

* Legal restrictions
* Resource limitations
* Political resistance
* Technical bottlenecks
* Strategic deterrents

---

## 8. FORWARD PROJECTION

Project system trajectory:

* CONTINUATION PATH → if current conditions hold
* ESCALATION PATH → if pressure increases
* STABILIZATION PATH → if counterforces act
* FAILURE PATH → if systems break

---

## 9. WATCHPOINTS

Define what must be monitored next:

* Specific triggers
* Policy decisions
* Infrastructure changes
* Behavioral shifts
* Data signals

These must be actionable and observable.
