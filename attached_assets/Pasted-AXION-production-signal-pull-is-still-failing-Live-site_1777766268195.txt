==================================================
OPTION A — CLIENT-SIDE MULTI-PROXY RESILIENT INGESTION
==================================================

Because EdgeOne Pages is static, make AXION work WITHOUT server.mjs.

Rewrite client-side signal fetching so it does not depend on /api/proxy/rss.

Requirements:
- Remove dependency on Node proxy for production success.
- Keep /api/proxy/rss only as optional if available.
- Add multiple public CORS/RSS fallback strategies.
- Do not allow all 78 feeds to hit one proxy at once.
- Use rotating proxy strategy.
- Batch feeds slowly.
- Retry failed feeds.
- Cap concurrent requests.

Fetch strategy order per feed:

1. Try rss2json:
https://api.rss2json.com/v1/api.json?rss_url=ENCODED_URL

2. If rss2json fails, try AllOrigins GET:
https://api.allorigins.win/get?url=ENCODED_URL

3. If that fails, try AllOrigins RAW:
https://api.allorigins.win/raw?url=ENCODED_URL

4. If that fails, try Jina AI Reader if suitable:
https://r.jina.ai/http://r.jina.ai/http://...
or use r.jina.ai/http:// + feed URL only if parsing returns usable XML/text.

5. If that fails, skip feed and record failure.

IMPORTANT:
Do NOT hit allorigins 78 times at once.
That is why production collapses.

Implement:
- max concurrency: 5
- per-feed timeout: 8 seconds
- retry count: 1
- stagger delay between batches: 500–1000ms
- proxy rotation
- fail fast on bad feeds
- continue if feed fails

Expected:
Even if each feed returns 10–15 items, 70+ feeds should produce hundreds of raw items.
